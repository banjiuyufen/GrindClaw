# 🦞 龙虾大赛 - Multi-Agent 科研协作系统实施方案

## 📖 项目概述
本项目基于 **OpenClaw** 框架为本人构建了一个虚拟科研团队，通过三位 Agent 的分工协作，覆盖本人部分科研流程。该系统旨在通过 Multi-Agent 协作实现科研任务自动化，让本人从繁琐的调研、编码与撰写中解放出来。

**核心愿景**：让每位科研工作者拥有一个 24 小时在线的虚拟科研团队。
**团队名称**：黑心小作坊（成员：墨墨、茶茶、码码）。
**当前版本**：v2.0 (2026-03-18)。

---

## 🛠️ 已实现的核心系统

### 1. 多层记忆系统 (memclawz)
解决 AI Agent 跨 Session 失忆的问题，确保科研上下文的连续性。

| 层级 | 访问速度 | 存储位置 | 用途 |
| :--- | :--- | :--- | :--- |
| **QMD** | < 1ms  | `memory/qmd/current.json`  | 工作记忆  |
| **Zvec** | < 10ms  | HNSW 向量索引  | 快速检索  |
| **MEMORY.md** | ~ 50ms  | 工作区根目录  | 长期精华知识  |

* **技术亮点**：
    * **身份连续性**：启动时自动加载 `SOUL.md` 以恢复 Agent 人格 。
    * **分类存储**：区分私聊与群聊记录，支持话题持续追踪 。
    * **自动归档**：每日凌晨 `02:00` 自动备份至日期命名的 Markdown 文件。

### 2. Agent 调度系统 [cite: 20]
采用三 Agent 协作架构，实现复杂任务的自动化拆解与执行。

| Agent | 角色 | 核心能力 | 飞书账号 |
| :--- | :--- | :--- | :--- |
| **墨墨** | **Coordinator (主管)**  | 任务规划、灵感解析、监督协调  | default  |
| **茶茶** | **Research Worker**  | 文献调研、竞品分析、报告撰写  | chacha  |
| **码码** | **Coding Worker**  | 代码实现、脚本编写、自动化  | mama  |

* **核心功能**：支持群聊任务分发 [@ 方式]、消息智能中转（Message Bus） 以及基于 `chat_id` 的双向通信协议。

### 3. AI vs AI 审稿系统
模拟多审稿人视角，提供高质量的论文预审意见。

* **角色设定**：由墨墨担任 AC（统筹），茶茶担任建设型评审（创新性），码码担任严苛型评审（技术细节）。
* **产出内容**：涵盖论文总结、优劣势分析、多项评分（1-5 分）及 Meta-Review 建议。

### 4. 自动化日报系统
* **早浪简报 (09:00)**：基于 92 个高质量 RSS 源自动生成中文 AI 生态要点。
* **ArXiv 播客 (10:40)**：自动筛选 Top 5 论文并使用 Edge-TTS 生成语音播客（和本人开源的arxiv-agent形成联动）。

## 📈 应用效果对比 (以文献调研为例)

| 对比项 | 传统方式  | 本系统方案  |
| :--- | :--- | :--- |
| **时间成本** | 约 3 天 | **30 分钟** |
| **执行流程** | 手动搜索、下载、阅读、整理 | 一键 @茶茶 自动输出报告 |
| **最终效果** | 信息分散，难以全面 | 结构化清晰，附带精确引用 |

> **案例**：在 EditDNA 项目中，茶茶 30 分钟内生成了 13 篇关键论文列表，并给出了 4.3/5 的创新性评估。

## 🛠️ 主要技术
1.  **飞书文档双写机制**：创建云端文档时同步进行本地备份，防止信息丢失。
2.  **独立身份标识**：各 Agent 拥有独立飞书 Bot 账号，提供真人的团队协作感。
3.  **多层记忆系统**：不同的 Agent 有独立的记忆分区，记忆系统三层分级。
4.  **联动 arXiv-agent**：告别繁琐爆炸的每日刷 arXiv，直接由 Agent 转换成最符合用户研究方向的短时播客和专属记忆。
